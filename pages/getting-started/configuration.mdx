# Configuration

Main environment variables and options for the Survey Platform.

## Environment file

Copy `.env.example` to `.env` and adjust values. Key variables:

### App

| Variable       | Description |
|----------------|-------------|
| `APP_NAME`     | Application name. |
| `APP_ENV`      | `local`, `production`, etc. |
| `APP_KEY`      | Laravel encryption key (`php artisan key:generate`). |
| `APP_DEBUG`    | `true` / `false`. |
| `APP_URL`      | Full URL of the app (e.g. `https://survey.example.com`). Used for links in emails and redirects. |
| `APP_LOCALE`   | Default locale (e.g. `en`). |
| `APP_FALLBACK_LOCALE` | Fallback locale. |

### Database

Default is SQLite. For MySQL/PostgreSQL, set:

- `DB_CONNECTION`, `DB_HOST`, `DB_PORT`, `DB_DATABASE`, `DB_USERNAME`, `DB_PASSWORD`

### Session & cache

- `SESSION_DRIVER` – e.g. `database`, `file`.
- `CACHE_STORE` – e.g. `database`, `file`, `redis`.
- `QUEUE_CONNECTION` – e.g. `database`, `redis` (if you use queues).

### Frontend (Vite)

The React app calls the API using `VITE_API_URL` (from Vite’s `import.meta.env`):

- **Development**: Often same origin (e.g. proxy) or set `VITE_API_URL=http://localhost:8000/api` so the SPA talks to Laravel.
- **Production**: Set `VITE_API_URL` to your API base URL (e.g. `https://survey.example.com/api`) before running `npm run build`, or leave unset if the app is served from the same host as the API.

### Mail (default / fallback)

- `MAIL_MAILER`, `MAIL_HOST`, `MAIL_PORT`, `MAIL_USERNAME`, `MAIL_PASSWORD`, `MAIL_FROM_ADDRESS`, `MAIL_FROM_NAME`

Survey notification and respondent emails use **per-survey SMTP** (configured in the dashboard under each survey’s email settings). The `.env` mail config is only used when no survey-specific SMTP is set (e.g. respondent summary sent via default mailer).

### Optional: translations (DeepL)

If the app uses DeepL for auto-translation:

- `DEEPL_API_KEY`
- Optional: `DEEPL_*` tuning (e.g. delay between languages) in `config/services.php`.

### Optional: media / storage

- `FILESYSTEM_DISK` – local, S3, etc.
- For S3: `AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`, `AWS_DEFAULT_REGION`, `AWS_BUCKET`.

## After changing .env

- Run `php artisan config:cache` in production (or `php artisan config:clear` to reload).
- Rebuild the frontend if you changed `VITE_*`: `npm run build`.
